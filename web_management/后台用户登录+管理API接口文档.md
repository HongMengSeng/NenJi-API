# 后台用户登录+管理 API 接口文档

本文档用于后台管理系统用户管理，包含后台普通员工（后台）和普通用户（小程序、客户端）的统一管理。

## 一、接口拆分原则

- 用户管理：员工和用户统一走 `/api/user/*`
- 员工和用户通过 `type` 字段筛选

## 二、时间格式约定

所有时间字段统一使用 `yyyy-MM-dd HH:mm` 格式。

- 正确示例：`2026-04-13 09:30`
- 正确示例：`2026-12-01 17:05`
- 不再使用：`2026/04/13 09:30`
- 不再使用：`2026-4-13 09:30`

## 二、公共响应结构

| 字段        | 类型                    | 说明             |
| --------- | --------------------- | -------------- |
| `code`    | Number                | 状态码，`200` 表示成功 |
| `message` | String                | 提示文案           |
| `data`    | Object / Array / Null | 业务数据           |

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

## 三、用户登录

### 3.1 接口信息

| 项目   | 内容                     |
| ---- | ---------------------- |
| 接口名称 | 用户登录                 |
| 请求路径 | `/api/user/login` |
| 请求方式 | `POST`                 |
| 请求格式 | `application/json`     |
| 接口说明 | 管理员登录（仅管理员可登录）        |

### 3.2 请求参数

| 字段         | 是否必传 | 类型     | 示例           | 说明     |
| ---------- | ---- | ------ | ------------ | ------ |
| `account` | 是    | String | `staff10001` | 登录账号 |
| `password` | 是    | String | `123456`     | 登录密码   |

### 3.3 请求示例

```json
{
  "account": "staff10001",
  "password": "123456"
}
```

### 3.4 成功响应示例

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": "staff_10001",
    "account": "staff10001",
    "role": "管理员",
    "loginTime": "2026-04-13 09:30",
    "token": "user-token-20260413-xxxx"
  }
}
```

### 3.5 失败场景

| 场景    | 状态码   | 提示文案         |
| ----- | ----- | ------------ |
| 账号为空  | `400` | 账号不能为空       |
| 密码为空  | `400` | 密码不能为空       |
| 账号不存在 | `401` | 账号不存在        |
| 密码错误  | `401` | 密码错误，请重新输入   |
| 账号禁用  | `403` | 账号已禁用，请联系管理员 |
| 非管理员  | `403` | 无权限，请使用管理员账号登录 |

### 4.5 用户字段说明

#### 员工字段

| 字段         | 类型     | 说明                   |
| ---------- | ------ | -------------------- |
| `id`       | String | 员工ID                 |

| `avatar`   | String | 头像，默认"无"              |
| `role`     | String | 角色，普通员工        |
| `status`   | String | 状态，启用/禁用            |
| `loginTime` | String | 最后登录时间              |

#### 用户字段

| 字段         | 类型     | 说明                   |
| ---------- | ------ | -------------------- |
| `id`       | String | 用户ID（微信授权后获得）       |
| `phone`    | String | 手机号                 |
| `account`  | String | 账号/昵称                   |
| `avatar`   | String | 头像URL                |
| `role`     | String | 固定为`普通用户`            |
| `status`   | String | 状态，启用/禁用            |
| `loginTime` | String | 最后登录时间              |

## 十、通用请求头

后台管理接口建议统一要求以下请求头：

| 字段             | 是否必传 | 类型     | 说明                     |
| -------------- | ---- | ------ | ---------------------- |
| `Content-Type` | 是    | String | 固定为 `application/json` |
| `token`        | 是    | String | 登录接口返回的令牌              |

## 十一、通用失败场景

| 场景          | 状态码   | 提示文案         |
| ----------- | ----- | ------------ |
| token 无效或过期 | `401` | 登录已过期，请重新登录  |
| 权限不足        | `403` | 权限不足，仅管理员可操作 |
| 参数错误        | `400` | 请求参数不完整或格式错误 |
| 数据不存在       | `404` | 用户不存在        |
| 服务异常        | `500` | 服务器异常，请稍后重试  |

## 十二、落地建议

- 所有接口统一走 `/api/user/*`
- 员工和用户通过 `type` 参数筛选
- 管理员不显示在用户列表中
- 文档、接口、前端字段都统一使用 `yyyy-MM-dd HH:mm` 时间格式
- status字段只有加载列表时显示
- 员工没有手机号和头像时显示"无"
