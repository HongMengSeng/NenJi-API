## 能记农场小程序后端本次修改说明

- **登录相关（`AuthController`）**
  - 按照《API 需求文档》实现了三种登录方式：
    - 一键登录：`POST /api/auth/login`，使用 `deviceId` 作为设备标识，在 `user` 表中创建或查找对应用户，返回 `token` 与 `userInfo`。
    - 微信登录：`POST /api/auth/wechat`，通过 `code` 调用微信 `jscode2session` 接口获取 `openid`，并与 `user` 表中的 `wx_openid` 关联，返回 `token` 与昵称 / 头像占位信息。
    - 手机号登录：`POST /api/auth/phone`，根据手机号查找或创建用户，模拟验证码校验逻辑，返回 `token` 与用户信息。
  - 继续保留登出与登录状态检查接口：
    - `POST /api/auth/logout`：无状态登出，前端删除本地 `token` 即可。
    - `GET /api/auth/check`：返回固定已登录状态与示例用户信息，便于前端联调。

- **数据库与图片存储（`mall_dining_system.sql` / `commodity` 表）**
  - 在 `commodity` 表中新增 `image_data` 列（`longblob`），用于存储商品图片的二进制数据，与 `ApplicationDbContext` 映射保持一致。
  - 在 SQL 结尾新增测试数据：
    - 插入两条分类数据（“新鲜蔬菜”“优选水果”）。
    - 插入三条商品数据（有机生菜、甜糯玉米、农家西红柿），包含 `image_url` 字段，便于通过 API 返回商品图片地址。

- **图片访问链路说明**
  - 前端可通过：
    - `CommodityController` 的列表、详情接口获取商品，并从 `image` 字段拿到图片访问地址（内部会映射到 `/api/commodity/{id}/image`）。
    - `GetImage` 接口优先返回数据库中的二进制图片（`image_data`），若为空则 302 跳转至 `image_url`。

- **注意事项**
  - 目前微信昵称与头像仍为占位实现，真实项目需要在微信登录后解密 `encryptedData` 来获取用户资料。
  - 登录产生的 `token` 为简单 Base64 字符串，仅用于联调，请在生产环境替换为安全的 JWT 或其他认证方案。
